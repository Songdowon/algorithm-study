# 🧩 BOJ 10798 - 세로읽기

## 📌 문제 정보
- **난이도**: Bronze 1
- **링크**: https://www.acmicpc.net/problem/10798

---

## 📝 문제 요약
- 총 5개의 문자열이 주어진다
- 문자열들을 **가로로 놓았다고 생각한 뒤**, 세로 방향으로 읽고 출력한다
- 짧은 문자열 때문에 없는 문자는 건너뛰고, 존재하는 문자만 이어 붙인다

---

## 💡 핵심 아이디어
- 문자열 5개를 리스트에 저장한다
- 가장 긴 문자열의 길이를 기준으로 열(`col`)을 순회한다
- 각 열마다 5개의 문자열을 위에서 아래로 확인한다
- 해당 위치의 문자가 존재할 때만 결과에 추가한다

---

## 🧠 접근 과정
- 처음 생각:
  - 문자열을 행처럼 두고 세로 방향으로 읽으면 된다
- 핵심:
  - 바깥쪽 반복문은 `열`
  - 안쪽 반복문은 `행`
- 주의:
  - 모든 문자열 길이가 같지 않기 때문에 인덱스 범위를 먼저 확인해야 한다
- 해결:
  - `if col < len(words[row])` 조건으로 문자가 있는 경우만 결과에 넣는다

---

## 🛠 사용한 것
- 문자열 리스트
- 이중 `for`문
- 리스트에 `append()`
- `''.join()`
- `list comprehension`
- 문자열 메서드 `ljust()`

---

## ✅ 내가 배운 점
- 문자열도 리스트처럼 인덱스로 접근할 수 있어서, 문자 하나씩 꺼내며 문제를 풀 수 있다
- 평소처럼 `행 기준`으로 읽는 게 아니라, **열 기준**으로 문제를 접근하는 방법을 배웠다
- 모든 문자열의 길이가 같지 않기 때문에, **해당 인덱스가 존재하는지 먼저 확인해야 한다**
- 처음에는 2차원 리스트처럼 보이지 않았지만, 여러 문자열을 모아 두고 보면 **행과 열 구조처럼 해석할 수 있다**는 것을 배웠다
- 결과를 문자열에 계속 더하는 방법도 가능하지만, 문자를 리스트에 모은 뒤 `''.join()`으로 합치는 방식이 더 깔끔하다는 것을 배웠다
- 같은 문제도 단순한 이중 `for`문, `list comprehension`, 문자열 패딩처럼 여러 방식으로 구현할 수 있어서 **문제를 보는 관점이 조금 넓어졌다**
- `ljust()`를 사용하면 문자열 길이를 맞추는 **패딩 방식의 처리 방법**으로도 문제를 풀 수 있다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 이중 `for`문 ✅ | 가장 기본적이고 직관적 |
| `list comprehension` | 코드가 짧고 간결함 |
| 문자열 패딩 (`ljust`) | 길이를 맞춰 처리할 수 있어 아이디어가 흥미로움 |
