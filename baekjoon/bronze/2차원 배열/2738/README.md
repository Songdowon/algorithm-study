# 🧩 BOJ 2738 - 행렬 덧셈

## 📌 문제 정보
- **난이도**: Bronze 3
- **링크**: https://www.acmicpc.net/problem/2738

---

## 📝 문제 요약
- 크기가 같은 두 행렬 `A`, `B`가 주어진다
- 같은 위치의 원소끼리 더한 결과 행렬을 출력한다
- 행과 열을 기준으로 각 원소를 순서대로 탐색하면 된다

---

## 💡 핵심 아이디어
- 행렬은 **2차원 리스트**로 입력받는다
- `A[i][j] + B[i][j]` 형태로 같은 위치의 값을 더한다
- 결과를 새로운 행렬에 저장한 뒤 한 줄씩 출력한다

---

## 🧠 접근 과정
- 처음 생각:
  - 두 행렬을 각각 입력받고 같은 위치끼리 더하면 된다
- 핵심:
  - 바깥쪽 반복문은 `행`
  - 안쪽 반복문은 `열`
- 해결:
  - 이중 `for`문으로 각 위치를 순회하면서 합을 구한다

---

## 🛠 사용한 것
- 2차원 리스트 `[][]`
- `for`문
- 리스트에 `append()`
- 언패킹 출력 `print(*row)`
- `list comprehension`
- `zip()`

---

## ✅ 내가 배운 점
- 행렬은 **리스트 안에 리스트가 들어간 2차원 구조**로 표현할 수 있다
- 2차원 데이터를 다룰 때는 **행 인덱스와 열 인덱스**를 나누어 생각해야 한다
- 출력할 때 `print(*row)`를 사용하면 리스트의 요소를 공백 기준으로 깔끔하게 출력할 수 있다
- `list comprehension`을 사용하면 리스트를 더 짧고 간결하게 만들 수 있다
- `zip()`은 여러 **반복 가능한 객체의 같은 위치 요소를 묶어주는 함수**로,
  같은 위치의 행 또는 원소를 함께 처리할 수 있어 코드가 더 간결해진다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 이중 `for`문 ✅ | 가장 기본적이고 직관적 |
| `list comprehension` | 코드가 짧고 파이썬스러움 |
| `zip()` 활용 | 같은 위치 데이터 묶기에 편리 |
| `for` + 출력 분리 | 구조를 이해하기 좋음 |
