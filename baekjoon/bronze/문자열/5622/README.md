# 🧩 BOJ 5622 - 다이얼

## 📌 문제 정보
- **난이도**: Bronze 2  
- **링크**: https://www.acmicpc.net/problem/5622  

---

## 📝 문제 요약
- 알파벳 문자열이 주어진다  
- 각 알파벳을 전화 다이얼 숫자로 바꾸고 걸리는 시간을 계산한다  
- 숫자 `N`을 누르는 데 걸리는 시간은 `N + 1`초이다  

---

## 💡 핵심 아이디어
- 이 문제는 **문자 → 숫자 매핑** 문제이다  
- 알파벳과 그것에 해당하는 숫자를 반환해 문제를 해결한다

---

## 🧠 접근 과정
- 처음에는 알파벳마다 숫자를 하나하나 대응시키면 문제는 해결할 수 있다고 생각했다  
- 하지만 이 방법은 코드가 길어지고 반복적이라고 판단했다
- 같은 숫자를 반환하는 알파벳들을 그룹으로 묶어서 처리해야겠다고 생각했다 

---

## 🛠 사용한 것
- 리스트 / dict 
- `ord()`  
- 문자열 인덱싱  
- 반복문 (`for`)  

---

## ⚠️ 주의할 점
- 문제에서 요구하는 값은 **다이얼 숫자 + 1초**이다  
- 알파벳을 잘못된 그룹에 넣으면 전체 결과가 틀릴 수 있다  
- 입력이 소문자로 들어올 수 있으므로  
  `input().upper()` 또는 `input().lower()`로 통일하는 것이 안전하다

---

## 🔍 풀이 방법 비교

### 1. 그룹 문자열 방식 (사용한 방법) ✅
- **아이디어**: `"ABC", "DEF", ...` 형태로 묶어서 해당 그룹 찾기  
- **장점**: 직관적이고 흐름이 잘 보임  
- **단점**: `in` 연산으로 매번 탐색하므로 약간 비효율적  

---

### 2. dictionary 방식
- **아이디어**: 문자 → 시간 값을 미리 저장  
- **장점**: 빠르고 명확함  
- **단점**: 값을 하나씩 직접 작성해야 함  

---

### 3. 아스키코드 + 문자열 인덱싱
- **아이디어**: `ord()`로 인덱스 변환 후 문자열에서 값 가져오기  
- **장점**: 코드가 짧고 가장 효율적  
- **단점**: 처음 보면 직관성이 떨어질 수 있음 

---

### 4. 문자열 압축 매핑
- **아이디어**: 다이얼 규칙을 하나의 문자열로 압축하여 인덱스로 접근  
- **장점**: 코드가 매우 간결하고 빠름  
- **단점**: 문자열 구조를 이해해야 함  

---

## 📚 배운 점
- **아스키코드(`ord`)란 문자에 대응되는 정수값이다**
  - `'A' → 65`, `'B' → 66`, ..., `'Z' → 90`
  - `'a' → 97`, `'z' → 122`
- 값을 정확히 기억하지 못해도 아래와 같이 활용할 수 있다 
  `ord(ch) - ord('A')` → `0 ~ 25`  
  👉 알파벳을 배열 인덱스로 변환할 수 있다  
- 문자열을 이용해 숫자를 직접 대응시키는 방식은  
  → 코드를 더 간결하게 만들 수 있다  
- `input().lower()` 또는 `input().upper()`를 사용하면  
  → 입력 형태에 상관없이 안정적으로 처리할 수 있다
