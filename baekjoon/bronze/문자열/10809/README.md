# 🧩 BOJ 10809 - 알파벳 찾기

## 📌 문제 정보
- **난이도**: Bronze 2
- **링크**: https://www.acmicpc.net/problem/10809

---

## 📝 문제 요약
- 소문자로 이루어진 단어가 하나 주어진다
- `a`부터 `z`까지 각 알파벳이 처음 등장한 위치를 출력한다
- 문자열에 해당 알파벳이 없으면 `-1`을 출력한다

---

## 💡 핵심 아이디어
- 알파벳 26개 각각에 대해 처음 등장한 위치를 찾아야 한다
- 각 알파벳을 기준으로 문자열을 탐색하거나,
  문자열을 한 번 순회하면서 위치를 기록하는 방식으로도 해결할 수 있다
- 문자를 인덱스로 변환하여 리스트에 저장하는 방식도 가능하다

---

## 🧠 접근 과정
- 처음에는 각 알파벳마다 `find()`로 위치를 찾는 방식을 떠올림
- 이 과정에서 알파벳 문자열을 직접 만들어 사용하면 된다는 것을 알게 됨
- 이후 문자열을 한 번만 순회하면서 인덱스를 기록하는 방식도 가능하다고 생각함
- `enumerate()`를 활용하면 인덱스와 문자를 동시에 다룰 수 있어 다양한 방식으로 접근할 수 있었다

---

## 🛠 사용한 것
- 문자열 (`str`)
- 리스트 (`list`)
- 딕셔너리 (`dictionary`)
- 문자열 메서드 `find()`
- 아스키 코드 변환 `ord()`
- 반복문 (`for`)
- `enumerate()`
- 출력 형식 `print(*result)`

---

## ⚠️ 주의할 점
- 문제는 **처음 등장한 위치**를 구하는 것이다
- 이미 기록된 문자는 다시 덮어쓰면 안 된다
- 알파벳 순서는 반드시 `a`부터 `z`까지 유지해야 한다
- 문자열 인덱스는 `0`부터 시작한다

---

## 🔍 풀이 방법 비교

### 1. `find()` 사용 ✅
- **아이디어**: 각 알파벳에 대해 `word.find(ch)` 호출
- **장점**: 가장 간단하고 직관적임
- **단점**: 알파벳마다 문자열을 반복해서 탐색함

---

### 2. 리스트 + 직접 기록
- **아이디어**: 길이 26의 리스트를 만들고, 문자열을 순회하며 처음 위치 저장
- **장점**: 문자열을 한 번만 순회해서 해결 가능
- **단점**: `ord()`를 이용한 인덱스 변환이 필요함

---

### 3. dictionary
- **아이디어**: 각 문자의 첫 등장 위치를 딕셔너리에 저장
- **장점**: “문자 → 위치” 대응 관계가 명확하게 보임
- **단점**: 마지막에 `a ~ z` 순서대로 다시 꺼내야 함

---

## ✅ 선택한 방법
- 이 문제는 구현이 단순하고 가독성이 중요하므로  
  **`find()`를 사용하는 방식이 가장 직관적이고 적절하다고 판단했다**

---

## 📚 배운 점
- 이 문제는 `find()`를 사용하면 가장 간단하고 직관적으로 해결할 수 있다
- 같은 문제라도 리스트를 사용해 직접 위치를 기록하는 방식으로도 구현할 수 있다
- 리스트 방식은 `ord()`를 이용한 문자 → 인덱스 변환 패턴을 연습하기에 좋다
- `ord()`를 이용하면 문자를 아스키 코드(정수)로 변환할 수 있다  
  예를 들어 `'A' → 65`, `'z' → 122`이며, 소문자와 대문자의 차이는 32이다
- 따라서 `ord(ch) - ord('a')`와 같이 계산하면
  `'a' ~ 'z'`를 `0 ~ 25`의 인덱스로 변환할 수 있다
- 딕셔너리 방식도 가능하지만, 이 문제처럼 알파벳 범위가 고정된 경우에는 `find()`나 리스트 방식이 더 자연스럽다고 느꼈다
