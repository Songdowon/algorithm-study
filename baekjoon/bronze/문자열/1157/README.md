# 🧩 BOJ 1157 - 단어 공부

## 📌 문제 정보
- **난이도**: Bronze 1
- **링크**: https://www.acmicpc.net/problem/1157

---

## 📝 문제 요약
- 알파벳 단어가 주어진다
- 이 단어에서 **가장 많이 사용된 알파벳**을 출력하는 문제이다
- 단, 가장 많이 사용된 알파벳이 **여러 개라면 `?`** 를 출력한다
- 대소문자는 구분하지 않는다

---

## 💡 핵심 아이디어
- 먼저 입력 문자열을 모두 **대문자**로 통일한다
- 각 알파벳이 몇 번 나왔는지 **개수를 세어야 한다**
- 가장 큰 빈도수를 찾은 뒤,
  - 그 값을 가진 알파벳이 **1개면 그 문자 출력**
  - **2개 이상이면 `?` 출력**

---

## 🧠 접근 과정
- 처음 생각:
  - 단어를 하나씩 보면서 각 문자의 등장 횟수를 저장해야겠다고 생각했다
- 핵심 처리:
  - 대소문자 구분이 없으므로 `upper()` 로 통일
  - `dictionary`를 이용해 문자별 빈도수 저장
- 마지막 판별:
  - 최댓값을 가지는 문자가 하나인지 여러 개인지 확인

---

## 🛠 사용한 것
- `list`, `dict`, `set`  
  → 다양한 자료구조로 문자 빈도수를 계산하는 방법 연습
- `strip().upper()`  
  → 입력 문자열 전처리 (공백 제거 + 대소문자 통일)
- `collections.Counter`  
  → 문자 등장 횟수를 간단하게 계산하는 내장 모듈
- 배열 기반 카운팅 (A-Z 인덱싱)  
  → `list[26]`을 활용한 빠른 빈도수 처리
- 아스키 코드 `ord()`  
  → 문자 → 숫자 인덱스로 변환하여 효율적인 접근 구현


---

## ⚠️ 주의할 점
- 입력은 대소문자가 섞여 들어올 수 있으므로  
  먼저 `upper()` 또는 `lower()`로 통일해야 한다
- 가장 많이 나온 알파벳이 **여러 개**일 수도 있으므로  
  최댓값만 찾고 끝내면 안 되고, 개수까지 확인해야 한다
- 문자 개수를 셀 때는 단순 출력 문제가 아니라  
  **빈도수 저장 구조**를 정확히 잡는 것이 중요하다

---

## 🚀 풀이 전략 선택 이유

이 문제는 "문자 빈도수 계산"이 핵심이므로  
여러 가지 접근 방법이 가능하다:

1. `dict` 기반 카운팅
2. `set + count()` 방식
3. `list(26)` 배열 인덱싱
4. `Counter` 모듈

이 중에서 **기본 풀이로 `dict`를 선택한 이유**는 다음과 같다:

- 가장 직관적인 방식 (가독성 ↑)
- 어떤 문자든 바로 key로 사용할 수 있음
- 알고리즘 구조를 명확하게 보여주기 좋음

---

## ⚖️ 풀이 방법 비교

| 방법 | 특징 | 시간 복잡도 | 장점 | 단점 |
|------|------|------------|------|------|
| `dict` | 문자 → 빈도수 저장 | O(N) | 직관적, 확장성 좋음 | 약간의 메모리 사용 |
| `set + count()` | 매번 문자열 탐색 | O(N²) | 코드 짧음 | 비효율적 |
| `list(26)` | A-Z 인덱스 매핑 | O(N) | 빠름, 메모리 효율 | 직관성 떨어짐 |
| `Counter` | 내장 모듈 | O(N) | 코드 매우 간단 | 내부 동작 이해 어려움 |

---

## ⚡ 성능 관점 정리

- 가장 비효율적인 방법:  
  👉 `set + count()` (문자마다 전체 문자열 다시 탐색)

- 가장 균형 잡힌 방법:  
  👉 `dict` (가독성 + 성능 둘 다 좋음)

- 가장 빠른 방법:  
  👉 `list(26)` (배열 기반, 인덱스 접근)

- 가장 간단한 방법:  
  👉 `Counter`

---

## 🎯 최종 선택

- 기본 풀이: `dict`
- 이유:
  - 문제 해결 흐름을 가장 잘 드러냄
  - 면접/설명 시 전달력이 좋음
  - 다른 자료구조로 확장하기 쉬움

---

## 🧠 확장 사고

이 문제는 단순 문자열 문제가 아니라  
다음과 같은 개념으로 확장된다:

- **빈도수 기반 문제 (Frequency Counting)**
- 해시 테이블 (`dict`) 활용
- 최댓값 탐색 + 중복 판별
- 데이터 전처리 (`upper()`)

---

## 📚 배운 점
- 문자열 문제에서도 단순 순회가 아니라  
  **빈도수 기반 사고 (counting)** 가 중요하다는 것을 이해했다
- 대소문자 처리를 먼저 해주면  
  이후 로직을 훨씬 단순하게 만들 수 있다
- `dict`, `list`, `set`, `Counter` 등  
  **여러 가지 방법으로 같은 문제를 해결할 수 있다는 것**을 경험했다
- 특히 `dict`는 직관적이고,  
  `list(26)` 방식은 성능적으로 더 효율적일 수 있다
- `Counter`는 빠르게 구현할 때 매우 강력하지만,  
  내부 동작을 이해하기 위해서는 직접 구현도 중요하다
- `ord(ch) - ord('A')` 방식은  
  **문자를 인덱스로 변환하는 대표적인 테크닉**이라는 것을 배웠다
- 최댓값을 찾는 문제에서는  
  **"값 비교"와 "중복 여부 판단"을 반드시 분리해서 생각해야 한다**
