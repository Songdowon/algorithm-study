# 🧩 BOJ 2720 - 세탁소 사장 동혁

## 📌 문제 정보
- **난이도**: Bronze 3
- **링크**: https://www.acmicpc.net/problem/2720

---

## 📝 문제 요약
- 거스름돈 `C`(센트 단위)가 주어진다
- 미국 동전 단위인
  - 쿼터(25센트)
  - 다임(10센트)
  - 니켈(5센트)
  - 페니(1센트)
  를 사용해서
- **동전 개수를 최소로 하면서** 각각 몇 개 필요한지 구하는 문제이다

---

## 💡 핵심 아이디어
- 큰 단위 동전부터 최대한 많이 사용하는 **그리디 방식**으로 해결할 수 있다
- 쿼터 개수를 구한 뒤 남은 금액에 대해 다임, 니켈, 페니 순서대로 처리하면 된다
- 이 문제는 **소수(달러 단위)**로 계산하지 말고, **정수(센트 단위)** 그대로 계산해야 한다

---

## 🧠 접근 과정
- 처음 생각:
  - 124센트라면 1.24달러로 바꿔서 계산해도 되지 않을까 생각할 수 있다
- 문제점:
  - `1.24`, `0.25`, `0.1`, `0.01` 같은 소수는 컴퓨터에서 정확하게 표현되지 않을 수 있다
  - 그래서 계산 과정에서 `0.04`가 아니라 `0.039999999...`처럼 저장될 수 있고,
    마지막 페니 계산이 틀어질 수 있다
- 해결:
  - 애초에 금액을 센트 단위 **정수 그대로** 두고
    `//`와 `%`를 사용해서 차례대로 동전 개수를 구한다

---

## 🛠 사용한 것
- 정수 나눗셈 `//`
- 나머지 연산 `%`
- `for`문
- 리스트
- 튜플 반환
- 언패킹(unpacking)
- `divmod()`

---

## ⚠️ 주의할 점
- 이 문제는 **소수로 계산하면 안 된다**
  - 예를 들어 `124 / 100 = 1.24`로 바꿔 계산하면
    부동소수점 오차 때문에 `penny`가 `4`가 아니라 `3.0`처럼 나올 수 있다
- 동전 개수는 정수이므로 **센트 단위 정수 연산**으로 처리하는 것이 안전하다
- `divmod(a, b)`는 `(a // b, a % b)`를 한 번에 구해주는 **파이썬 내장 함수**이다
- 여러 값을 반환할 때 `return a, b, c, d`처럼 쓰면 실제로는 **튜플**로 반환된다
- 언패킹할 때 오른쪽 값의 개수와 왼쪽 변수의 개수가 같아야 한다

---

## ✅ 배운 점
- 그리디 알고리즘은 **큰 단위부터 차례대로 선택해도 최적해가 보장되는 경우**에 사용할 수 있다
- 소수(부동소수점) 계산이 포함된 경우 오차가 발생할 수 있으므로
  가능한 **정수 단위**로 변환해서 계산하는 습관이 중요하다
- `//`는 몫, `%`는 나머지를 구하는 연산자이며
  함께 사용하면 동전 문제를 쉽게 해결할 수 있다
- `divmod()`를 사용하면 몫과 나머지를 동시에 구할 수 있어서 코드가 더 간결해진다
- 함수에서 여러 값을 반환할 때는 튜플 형태로 묶여서 반환되고,
  이를 변수 여러 개로 바로 받을 수 있다
- 여러 값을 한 번에 반환받아 각각의 변수에 순서대로 나누어 저장하는 것을 **언패킹**이라고 한다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 나눗셈 + 나머지 직접 갱신 ✅ | 가장 직관적이고 많이 쓰임 |
| 동전 배열 순회 | 코드 재사용성이 좋음 |
| `divmod()` 활용 | 몫과 나머지를 한 번에 처리해서 깔끔함 |
