# 🧩 BOJ 10810 - 공 넣기

## 📌 문제 정보
- **난이도**: Bronze 3
- **링크**: https://www.acmicpc.net/problem/10810

---

## 📝 문제 요약
- N개의 바구니가 있다 (초기값 0)
- M번의 입력이 주어진다
- 각 입력마다 i부터 j까지 바구니에 k번 공을 넣는다
- 만약 안에 공이 있으면 빼고 새로운 공을 넣는다
- 최종 바구니 상태를 출력한다
---

## 💡 핵심 아이디어
- **리스트를 생성하고 초기화**한 뒤,
  주어진 구간 `[i, j]`에 대해 값을 갱신하는 **구간 시뮬레이션 문제**
- 인덱스는 1부터 시작하므로 → 0-based index로 변환 필요(`i-1`) 

---

## 🧠 접근 과정
- 처음 생각: 
  - 바구니를 리스트로 만들고 직접 값 변경
- 막힌 부분:
  - 리스트를 초기화하지 않고 접근함
  - 출력이 리스트 형태로 그대로 나옴
- 해결 과정:
  - `basket = [0] * n` 으로 초기화한 뒤 반복문으로 구간 갱신 
  - `print(*basket)`으로 공백 기준 출력
---

## 🛠 사용한 것
- 리스트 초기화 (`[0] * n`)
- 반복문 (`for`)
- `map(int, input().split())`
- print(`*basket`)
- tuple

---

## ⚠️ 주의할 점
- 인덱스가 1부터 시작하므로  **반드시 `i-1`로 변환해야 함**
- `range(i - 1, j)` 형태로 사용해야 올바른 구간 처리 가능

---

## 📚 배운 점
- 리스트는 `[0] * n` 형태로 한번에 초기화할 수 있다.
- 리스트를 사용할 때는 **먼저 크기와 초기값을 설정하는 습관**이 중요하다
- 리스트를 이용한 **직관적인 시뮬레이션 방식**이 유용하다
- `print(*basket)`을 사용하면 리스트 형태가 아니라 원하는 출력 형식으로 바로 출력할 수 있다.
- 입력 데이터를 `(i, j, k)` 형태의 tuple로 묶어 관리할 수 있다.
- tuple을 사용하면 입력과 로직을 분리하여 코드 구조를 더 깔끔하게 만들 수 있다.
  또한 "하나의 의미 있는 데이터 묶음"을 표현할 때 적합하다
