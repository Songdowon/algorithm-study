# 🧩 BOJ 3052 - 나머지

## 📌 문제 정보
- **난이도**: Bronze 2
- **링크**: https://www.acmicpc.net/problem/3052

---

## 📝 문제 요약
- 10개의 정수를 입력받는다
- 각 수를 42로 나눈 나머지를 구한다
- 서로 다른 나머지의 개수를 출력한다

---

## 💡 핵심 아이디어
- 핵심은 “서로 다른 값의 개수”를 구하는 것
- 나머지를 구한 뒤 **중복을 제거**하면 된다
- 이 문제는 전형적인 **중복 제거 문제 → set 활용**이 핵심

---

## 🧠 접근 과정
- 처음에는 리스트에 나머지를 저장하고 중복을 직접 검사(`if not in`)하는 방법을 생각
- 하지만 매번 탐색이 필요해서 비효율적이라고 판단
- 이후 `set`을 사용하면 **자동으로 중복이 제거된다는 점**을 활용
- 가장 간단하고 효율적인 방식으로 해결 가능

---

## 🛠 사용한 것
- 리스트 (`list`)
- 집합 (`set`) ✅
- boolean 배열
- 반복문 (`for`)
- 집합 컴프리헨션

---

## ⚠️ 주의할 점
- 나머지는 항상 `0 ~ 41` 범위
- 입력 개수는 10개로 고정
- “서로 다른 값”이 핵심 → 중복 제거 필수

---

## 🔍 풀이 방법 비교

### 1. 리스트 + if문
- **아이디어**: 리스트에 저장하면서 중복이면 추가하지 않음
- **장점**: 동작 원리를 이해하기 쉬움
- **단점**: `in` 탐색이 반복됨 → 비효율적 (O(n²))

---

### 2. `set` ✅ (추천)
- **아이디어**: 나머지를 set에 넣어 중복 자동 제거
- **장점**
  - 코드가 매우 간결함
  - 중복 제거가 자동으로 처리됨
  - `in` 연산이 빠름 (O(1))
- **단점**: 없음 (이 문제에서는 최적)

---

### 3. list → set 변환
- **아이디어**: 리스트에 저장 후 마지막에 set으로 변환
- **장점**: 흐름이 명확 (저장 → 처리)
- **단점**: 불필요한 단계가 추가됨

---

### 4. boolean 배열
- **아이디어**: 나머지(0~41)를 인덱스로 사용하여 등장 여부 체크
- **장점**: 매우 빠르고 알고리즘적으로 깔끔
- **단점**: 범위를 알아야 함 / 직관성은 set보다 떨어질 수 있음

---

## ✅ 선택한 방법
- 이 문제는 **“중복 제거”가 핵심이므로 set이 가장 적합하다**
- 특히 파이썬에서는 set을 활용하면 **코드를 가장 간결하고 효율적으로 작성할 수 있다**

---

## 📚 배운 점
- “서로 다른 값의 개수” 문제는 **set을 먼저 떠올리는 것이 중요하다**
- `set`은 중복 제거뿐 아니라 빠른 탐색(O(1))이 가능하다
- 파이썬에서는 **문제의 의도에 맞는 자료구조 선택이 가장 중요하다**
-  `if __name__ == "__main__":` 구조를 사용하면  
  **코드를 모듈처럼 재사용하거나 테스트하기 쉬운 구조로 만들 수 있다**
