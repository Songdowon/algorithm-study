# 🧩 BOJ 5597 - 과제 안 내신 분..?

## 📌 문제 정보
- **난이도**: Bronze 3
- **링크**: https://www.acmicpc.net/problem/5597

---

## 📝 문제 요약
- 1번부터 30번까지 총 30명의 학생이 있다
- 이 중 28명의 학생이 과제를 제출한다
- 제출하지 않은 2명의 출석 번호를 오름차순으로 출력한다

---

## 💡 핵심 아이디어
- 전체 학생 번호는 `1 ~ 30`
- 제출 여부를 기록한 뒤, 제출하지 않은 번호를 찾는다
- “존재 여부 체크” 문제로 볼 수 있다

---

## 🧠 접근 과정
- 처음에는 리스트에서 제출한 학생을 제거하는 방식(`remove`)을 떠올림
- 하지만 탐색 비용이 발생하므로 다른 방법을 고민
- 이후 `set`, `boolean 배열` 방식으로 확장

---

## 🛠 사용한 것
- 리스트 (`list`)
- 집합 (`set`)
- boolean 배열
- 반복문 (`for`)
- 출력 형식 지정 `print(*result, sep="\n")`

---

## ⚠️ 주의할 점
- 학생 번호는 `1 ~ 30` 범위
- boolean 배열은 `check = [False] * 31`로 설정하면 인덱스와 번호를 맞출 수 있음
- 출력은 반드시 오름차순

---

## 🔍 풀이 방법 비교

### 1. 리스트 + `remove()`
- **아이디어**: 전체 리스트에서 제출한 학생 제거
- **장점**: 직관적이고 처음 떠올리기 쉬움
- **단점**: `remove()`는 탐색이 필요 → 비효율적 (O(n))

---

### 2. `set` (포함 여부 확인)
- **아이디어**: 제출한 학생을 집합으로 저장 후, 포함 여부 검사
- **장점**: `in` 연산이 빠르고 코드가 간결함
- **단점**: 순서를 유지하지 않기 때문에 출력 시 별도 처리 필요

---

### 3. `set - set` (차집합)
- **아이디어**: 전체 집합 - 제출 집합
- **장점**: 가장 간단하고 파이썬스럽게 해결 가능
- **단점**: 정렬(`sorted`) 필요

---

### 4. boolean 배열 ✅ (선택)
- **아이디어**: 인덱스를 학생 번호로 사용하여 제출 여부 체크
- **장점**: 구조적으로 문제와 가장 잘 맞고 직관적임
- **단점**: 번호 범위를 알아야 함

---

## ✅ 선택한 방법
- 이 문제는 “번호 기반 존재 여부 체크” 문제이므로  
  **boolean 배열 방식이 가장 직관적이고 적절하다고 판단했다**

---

## 📚 배운 점
- 같은 문제라도 자료구조에 따라 여러 방식으로 구현할 수 있다
- `remove()` 방식은 직관적이지만, 시간복잡도 측면에서는 좋은 선택이 아닐 수 있다
- `set`은 포함 여부 확인이 빠르기 때문에 탐색 문제에서 유용하다
- boolean 배열은 번호 체크 문제에서 가장 직관적이고 실전에서도 자주 쓰이는 방식이다
- `print(*result, sep="\n")`을 사용하면 여러 값을 줄바꿈 형태로 깔끔하게 출력할 수 있다
