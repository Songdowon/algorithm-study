# 🧩 BOJ 2501 - 약수 구하기

## 📌 문제 정보
- **난이도**: Bronze 3
- **링크**: https://www.acmicpc.net/problem/2501

---

## 📝 문제 요약
- 자연수 `N`의 약수를 작은 수부터 차례대로 나열한다
- 그중에서 `K`번째로 작은 약수를 구하는 문제이다
- 만약 `K`번째 약수가 존재하지 않으면 `0`을 출력한다

---

## 💡 핵심 아이디어
- 어떤 수 `i`가 `N`의 약수인지 확인하려면 나눴을 때 나머지가 0인지 검사하면 된다
- 따라서 `1`부터 `N`까지 차례대로 확인하면서 약수를 찾을 수 있다
- 약수를 찾을 때마다 개수를 세고, `K`번째 약수를 찾으면 바로 출력하면 된다
- 끝까지 확인했을 때 `K`번째 약수가 없으면 `0`을 출력한다

---

## 🧠 접근 과정
- 가장 먼저 떠올릴 수 있는 방법은 `1`부터 `N`까지 모든 수를 확인하는 것이다
- 각 수가 `N`을 나누어떨어지게 하면 그 수는 약수이다
- 약수를 찾을 때마다 순서를 세면 `K`번째 약수를 쉽게 찾을 수 있다
- 문제의 입력 범위가 크지 않기 때문에 이 방식만으로도 충분히 해결할 수 있다

---

## 🛠 사용한 것
- `for`문
- 나머지 연산 `%`
- 조건문 `if`
- 카운트 변수

---

## ⚠️ 주의할 점
- 약수를 찾는 순서는 **작은 수부터**여야 한다
- `K`번째 약수가 없을 수도 있으므로, 그런 경우에는 반드시 `0`을 출력해야 한다
- 약수를 전부 저장하지 않고, 찾는 즉시 개수를 세면서 처리해도 된다

---

## ✅ 배운 점
- 문제에서 원하는 것이 특정 순서의 값이라면, 조건을 만족할 때마다 **카운트를 세는 방식**이 유용하다
- 입력 범위가 크지 않은 문제는 복잡한 최적화보다 **정확하고 단순한 구현**이 더 중요할 수 있다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 전체 약수 저장 방식 | 약수를 리스트에 전부 저장한 뒤 `K`번째 값을 확인 |
| 카운트 방식 | 약수를 찾을 때마다 개수를 세고, `K`번째에서 바로 종료 |
| 리스트 컴프리헨션 방식 | 파이썬답게 짧게 작성할 수 있지만 본질은 전체 저장 방식과 같음 |
