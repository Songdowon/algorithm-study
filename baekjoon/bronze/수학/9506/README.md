# 🧩 BOJ 9506 - 약수들의 합

## 📌 문제 정보
- **난이도**: Bronze 1
- **링크**: https://www.acmicpc.net/problem/9506

---

## 📝 문제 요약
- 어떤 수 `n`이 주어졌을 때, 자기 자신을 제외한 약수들의 합이 `n`과 같으면 완전수로 판단한다
- 완전수이면 약수들의 합 형태로 출력하고, 아니면 `is NOT perfect.`를 출력한다
- 입력은 여러 줄로 주어지며 `-1`이 입력되면 종료한다

---

## 💡 핵심 아이디어
- `1`부터 `n-1`까지 확인하면서 `n`의 약수를 찾는다
- 약수들을 리스트에 저장하고, 동시에 합도 함께 구한다
- 약수들의 합이 `n`과 같으면 완전수이다
- 이 문제에서는 **정답 판별**뿐 아니라 **출력 형식을 정확히 맞추는 것**도 중요하다

---

## 🧠 접근 과정
- 처음에는 약수를 찾으면서 바로 출력하려고 했다
- 그런데 그렇게 하면 마지막 원소 뒤에도 `+`가 붙는 문제가 생긴다
- 그래서 약수들을 먼저 저장한 뒤, 나중에 원하는 형식에 맞게 출력하도록 바꾸었다
- 이 과정에서 같은 결과라도 문자열을 만드는 방식은 여러 가지가 있다는 점을 확인했다

---

## 🛠 사용한 것
- `while True`
- 약수 탐색
- 리스트 저장
- 누적합
- 문자열 출력 처리
- 조건문 / 인덱스 / 슬라이싱 / `join()`

---

## ✅ 내가 배운 점
- 출력 문제에서는 **값을 구하는 과정**뿐 아니라 **형식을 맞추는 과정**도 중요하다는 걸 느꼈다
- 여러 값을 이어서 출력할 때는 마지막 원소를 어떻게 처리할지 먼저 생각해야 한다
- 같은 결과를 만들더라도 구현 방식에 따라 코드의 길이와 가독성이 달라질 수 있다
- `join()`은 간단하고 깔끔하지만, 직접 구현해 보면서 마지막 원소 처리 원리를 더 분명하게 이해할 수 있었다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| `join()` 사용 ✅ | 가장 짧고 깔끔하게 표현 가능 |
| 마지막 인덱스를 조건문으로 처리 | 출력 원리를 직접 이해하기 좋음 |
| 슬라이싱으로 마지막 값 분리 | 구조가 분명해서 읽기 쉬움 |

---

## 🧩 출력 형식에서 집중해서 배운 점
- 반복문 안에서 바로 출력하면 마지막에도 `+`가 붙는 실수가 생기기 쉽다
- 그래서 출력할 때는 다음과 같은 여러 방식을 비교해 볼 수 있었다
  - 문자열을 한 번에 완성해서 출력하는 방법
  - 마지막 원소를 따로 처리하는 방법
  - `join()`으로 연결하는 방법
- 이번 문제를 통해 **출력 제어도 구현의 중요한 일부**라는 점을 배웠다
