# 🧩 BOJ 2745 - 진법 변환

## 📌 문제 정보
- **난이도**: Bronze 2
- **링크**: https://www.acmicpc.net/problem/2745

---

## 📝 문제 요약
- `B`진법으로 표현된 수 `N`이 주어진다
- 주어진 수를 **10진법 수로 변환**하여 출력하는 문제이다
- 숫자 `0~9`뿐 아니라 알파벳 `A~Z`도 사용하며
  `A=10`, `B=11`, ..., `Z=35`를 의미한다

---

## 💡 핵심 아이디어
- 문자열을 **왼쪽부터 한 글자씩 확인**하면서 값을 누적한다
- 현재까지의 결과에 `진법 B`를 곱하면 자릿수가 한 칸 올라간다
- 그 뒤 현재 문자의 값을 더하면 된다

---

## 🧠 접근 과정
- 처음 생각:
  - 진법 변환은 각 자리에 `진법의 거듭제곱`을 곱해서 더하는 방식이다
- 핵심:
  - 굳이 뒤에서부터 `B^0`, `B^1`, `B^2`를 직접 계산하지 않아도
    왼쪽부터 보면서 `이전 결과 × B + 현재 값`으로 처리할 수 있다
- 해결:
  - 문자 하나를 숫자값으로 바꾼 뒤 누적 계산한다

---

## 🛠 사용한 것
- 문자열 순회
- `for`문
- `enumerate()`
- `isdigit()`
- `ord()`
- `reversed()`
- 진법 누적 계산

---

## ⚠️ 주의할 점
- 알파벳은 `A=10`부터 시작한다
- 입력이 문자열이므로 바로 정수로 바꾸면 안 된다

---

## ✅ 배운 점
- 진법 변환은 단순히 외우는 것이 아니라  
  **이전 결과에 진법을 곱하고 현재 자리값을 더하는 방식**으로 이해할 수 있다
- 문자열로 입력된 숫자는 문자 단위로 순회하면서 처리할 수 있다
- `isdigit()`을 사용하면 숫자인지 아닌지를 쉽게 판별할 수 있다
- 같은 문제도 누적 계산, 거듭제곱 계산, `enumerate()` 활용처럼 여러 방식으로 구현할 수 있다
- `reversed()`를 사용하면 문자열이나 리스트를 **뒤에서부터 거꾸로 순회**할 수 있다
- `ord() - ord('A')`를 이용하면 알파벳을 숫자값으로 바꿀 수 있다
  - `ord()`는 흔히 아스키 코드라고 생각하기 쉽지만, 정확히는 **문자의 유니코드 코드 포인트를 반환하는 함수**라는 것을 알게 되었다
  - 영어 대문자, 소문자, 숫자는 **유니코드 값이 아스키 코드와 동일**하다
  - 자주 쓰는 대표적인 유니코드는 함께 기억해두면 편하다  
    - `'0'` = 48, `'9'` = 57  
    - `'A'` = 65, `'Z'` = 90  
    - `'a'` = 97, `'z'` = 122

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 누적 계산 ✅ | 가장 많이 쓰이고 가장 깔끔함 |
| 뒤에서부터 거듭제곱 계산 | 진법 원리를 이해하기 좋음 |
| `enumerate()` 활용 | 인덱스와 자리값을 함께 다루기 편함 |
