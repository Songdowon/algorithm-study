# 🧩 BOJ 11005 - 진법 변환 2

## 📌 문제 정보
- **난이도**: Bronze 1
- **링크**: https://www.acmicpc.net/problem/11005

---

## 📝 문제 요약
- 10진법 수 `N`과 진법 `B`가 주어진다
- `N`을 **B진법으로 변환한 결과**를 출력하는 문제이다
- 10 이상인 값은 알파벳 `A~Z`로 표현하며  
  `A=10`, `B=11`, ..., `Z=35`를 의미한다

---

## 💡 핵심 아이디어
- `N`을 `B`로 나눈 **나머지**가 현재 자리의 값이 된다
- 그 뒤 `N //= B`를 하면 다음 자리로 넘어갈 수 있다
- 이 과정을 `N`이 0이 될 때까지 반복하면 된다
- 나머지는 **낮은 자리부터** 나오므로, 저장한 뒤 뒤집어야 한다

---

## 🧠 접근 과정
- 처음 생각:
  - 10진수를 다른 진법으로 바꿀 때는 계속 나누면서 나머지를 구한다
- 핵심:
  - `N % B`로 현재 자릿값을 구하고
  - `N //= B`로 다음 자릿수 계산으로 넘어간다
- 해결:
  - 나머지를 하나씩 저장한 뒤, 마지막에 뒤집어서 출력한다

---

## 🛠 사용한 것
- `while`문
- 나머지 연산 `%`
- 몫 연산 `//`
- 리스트 `append()`
- `reversed()`
- `join()`
- `chr()`

---

## ⚠️ 주의할 점
- 나머지는 **뒤에서부터(낮은 자리부터)** 구해지므로 바로 출력하면 거꾸로 된다
- 10 이상은 숫자가 아니라 `A~Z`로 바꿔야 한다
- `n //= b`는 반드시 반복문 안에서 수행해야 한다

---

## ✅ 배운 점
- 10진수를 다른 진법으로 바꿀 때는  
  **나누기와 나머지 연산을 반복**해서 각 자릿값을 구할 수 있다
- `N % B`는 현재 자리의 값, `N // B`는 다음 자리로 넘어가는 역할을 한다
- 10 이상인 값은 `chr(값 - 10 + ord('A'))`로 문자로 바꿀 수 있다
- `chr()`은 정수 코드를 문자로 바꾸는 함수이고,  
  `ord()`는 문자를 정수 코드값으로 바꾸는 함수라는 점을 알게 되었다
- 나머지는 낮은 자리부터 나오기 때문에  
  **리스트에 저장한 뒤 뒤집어서 출력**해야 한다
- 문자열 앞에 계속 붙이는 방식도 가능하지만,  
  **리스트에 `append()`한 뒤 `''.join()`으로 합치는 방식이 더 효율적**이라는 점을 알게 되었다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 문자열 앞에 붙이기 | 가장 직관적이지만 문자열 누적은 비효율적일 수 있음 |
| 리스트에 저장 후 뒤집기 ✅ | 가장 많이 쓰이고 효율적인 방식 |
| 문자 테이블 활용 | 조건문 없이 더 깔끔하게 구현 가능 |
