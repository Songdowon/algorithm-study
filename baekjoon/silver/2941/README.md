# 🧩 BOJ 2941 - 크로아티아 알파벳

## 📌 문제 정보
- **난이도**: Silver 5
- **링크**: https://www.acmicpc.net/problem/2941

---

## 📝 문제 요약
- 알파벳 소문자와 몇 가지 **크로아티아 알파벳**으로 이루어진 문자열이 주어진다
- 아래 8개를 하나의 알파벳으로 센다
  - `c=`, `c-`, `dz=`, `d-`, `lj`, `nj`, `s=`, `z=`
- 문자열에 들어 있는 알파벳의 개수를 구해야 한다
- 여기서 `dz=`는 반드시 **3글자를 하나로 묶어서** 처리해야 한다

---

## 💡 핵심 아이디어
- 문자열에서 크로아티아 알파벳을 **하나의 문자로 치환**하면 된다
- 예를 들어 `*` 같은 한 글자로 바꾸면, 마지막 문자열 길이가 곧 정답이 된다
- 이때 `dz=`는 `z=`보다 길기 때문에 **먼저 처리해야 한다**
- 또는 문자열을 왼쪽부터 직접 탐색하면서  
  **3글자 → 2글자 → 1글자 순서**로 확인할 수도 있다

---

## 🧠 접근 과정
- 처음 생각:
  - 그냥 문자열 길이를 세면 될 것 같지만, 크로아티아 알파벳은 여러 글자가 하나의 문자로 취급된다
- 핵심:
  - 여러 글자를 하나의 문자처럼 처리해야 하므로, 치환하거나 직접 탐색해야 한다
- 주의:
  - `dz=`와 `z=`가 겹치므로, 짧은 패턴을 먼저 처리하면 잘못 셀 수 있다
- 해결:
  - `dz=`를 먼저 처리한 뒤 나머지 크로아티아 알파벳을 처리하거나
  - 매 위치마다 가장 긴 패턴부터 확인한다

---

## 🛠 사용한 것
- 문자열
- `replace()`
- 슬라이싱
- `while`문
- `set`, `list`

---

## ✅ 내가 배운 점
- 문자열 문제에서도 단순히 글자 하나씩 보는 것이 아니라, **여러 글자를 하나의 패턴으로 묶어 처리**해야 할 수 있다는 점을 배웠다
- `replace()`를 사용하면 특정 문자열을 한 글자로 치환해서 문제를 간단하게 해결할 수 있다
- 문자열을 직접 탐색할 때는 **긴 패턴을 먼저 확인해야 한다**는 점이 중요하다
  - 특히 `dz=`처럼 다른 패턴과 일부가 겹치는 경우에는 검사 순서가 정답에 큰 영향을 준다
- 슬라이싱을 이용하면 현재 위치에서 2글자, 3글자 패턴을 쉽게 확인할 수 있다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| `replace()`로 치환 ✅ | 가장 간단하고 직관적 |
| 직접 탐색 (`while`) | 문자열 처리 흐름을 이해하기 좋음 |
| 조건문으로 패턴 비교 | 동작 원리가 가장 잘 보임 |
