# 🧩 BOJ 1316 - 그룹 단어 체커

## 📌 문제 정보
- **난이도**: Silver 5
- **링크**: https://www.acmicpc.net/problem/1316

---

## 📝 문제 요약
- 여러 개의 단어가 주어진다
- 각 단어가 **그룹 단어**인지 판별해야 한다
- 그룹 단어란  
  **각 문자가 연속해서만 나타나는 단어**를 뜻한다
- 어떤 문자가 한 번 등장했다가 다른 문자가 나온 뒤,
  다시 등장하면 그룹 단어가 아니다

예시:
- `ccazzzzbb` → 그룹 단어
- `aabbbccb` → 그룹 단어 아님  
  (`b`가 등장했다가 `c` 뒤에 다시 나옴)

---

## 💡 핵심 아이디어
- 문자열을 왼쪽부터 하나씩 확인하면서,
  **이전 문자와 현재 문자**를 비교하면 된다
- 현재 문자가 이전 문자와 같다면
  연속 등장 중이므로 괜찮다
- 현재 문자가 이전 문자와 다른데,
  이미 전에 등장한 적이 있다면 그룹 단어가 아니다
- 따라서
  - 이전 문자 저장용 변수 `prev`
  - 이미 등장한 문자 저장용 `set`
  만 있으면 해결할 수 있다

---

## 🧠 접근 과정
- 처음 생각:
  입력받은 단어를 문자 하나씩 순회하면서 조건을 확인한다
- 핵심:
  현재 문자와 이전 문자를 비교했을 때 
  두 문자가 같으면 계속 진행하고 다르면 새 구간이 시작된 것으로 본다
  이때 현재 문자가 이미 등장했던 문자라면 한 번 끊겼다가 다시 나온 것이므로 그룹 단어가 아니다


---

## 🛠 사용한 것
- 문자열
- `for`문
- `set`
- 이전 문자와 현재 문자 비교

---

## ✅ 내가 배운 점
- 두 가지 상태만 구분하면 되는 경우에는
  `boolean`, `0과 1`, `set` 같은 방식을 활용해 상태를 표현하거나 관리하면 매우 효과적이라는 것을 다시 배웠다
- 슬라이싱을 활용하면 문자열을 문자 단위뿐만 아니라 구간이나 패턴 단위로 비교하는 방식도 가능하다고 생각해볼 수 있었다
- `set`을 사용하면 이미 등장한 문자인지 빠르게 확인할 수 있어서 문자열의 흐름을 효율적으로 판별할 수 있다
- 현재 문자와 이전 문자를 비교하기 위해 
  이전 상태를 따로 저장해 두는 방식이 중요하다는 점도 다시 확인할 수 있었다


---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 이전 문자 + `set` 사용 ✅ | 가장 많이 쓰이고 직관적 |
| 슬라이싱으로 이전 구간 확인 | 구현은 가능하지만 비효율적일 수 있음 |
| 압축 문자열 만들어 비교 | 아이디어는 재밌지만 조금 우회적 |
