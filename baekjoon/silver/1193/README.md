# 🧩 BOJ 1193 - 분수찾기

## 📌 문제 정보
- **난이도**: Silver 5
- **링크**: https://www.acmicpc.net/problem/1193

---

## 📝 문제 요약
- X번째 분수를 구하는 문제이다
- 분수들은 대각선 방향으로 일정한 규칙에 따라 나열된다

예시:
- 1번째: `1/1`
- 2번째: `1/2`
- 3번째: `2/1`
- 4번째: `3/1`

- 각 대각선에 포함된 분수의 개수는 1개, 2개, 3개, 4개 ... 식으로 늘어난다
- 대각선 번호가 홀수인지 짝수인지에 따라
  분자와 분모가 증가하는 방향이 달라진다

---

## 💡 핵심 아이디어
- 분수를 하나씩 직접 나열하기보다  
  **X가 몇 번째 대각선에 속하는지** 먼저 찾으면 된다
- 대각선의 길이는 1, 2, 3, 4 ... 로 증가하므로  
  `1 + 2 + 3 + ...` 형태로 누적하면서
  X가 포함되는 구간을 찾을 수 있다
- 그다음, 해당 대각선 안에서 X가 몇 번째 위치인지 구하면
  분자와 분모를 결정할 수 있다
- 이때 핵심은  
  **대각선 번호의 홀짝에 따라 이동 방향이 반대**라는 점이다

---

## 🧠 접근 과정
- 처음에는 분수를 직접 써 보면서 규칙을 찾았다
- `1/1` 다음에는 `1/2`, `2/1`
- 그다음은 `3/1`, `2/2`, `1/3`이 나온다
- 같은 대각선에 있는 분수들은
  `분자 + 분모`가 항상 일정하다는 특징이 있다
- 따라서 먼저 X가 속한 대각선을 찾고,
  그 안에서의 위치를 구한 뒤
  홀수 대각선인지 짝수 대각선인지에 따라
  분자와 분모를 정하면 된다

---

## 🛠 사용한 것
- `while`문
- 누적합
- 홀수/짝수 판별
- 규칙 기반 구현

---

## ✅ 내가 배운 점
- 규칙이 있는 나열 문제에서는
  값을 전부 직접 만들기보다  
  **구간 단위로 바라보는 방식**이 더 중요하다는 걸 느꼈다
- 누적합을 이용하면
  어떤 값이 어느 구간에 속하는지 효율적으로 찾을 수 있다는 점을 다시 확인했다
- 같은 규칙 안에서도
  홀수와 짝수에 따라 방향이 달라질 수 있으므로
  조건을 나누어 정리하는 습관이 중요하다고 느꼈다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 대각선 번호를 `while`로 찾기 ✅ | 가장 직관적이고 많이 쓰이는 방식 |
| 누적합의 시작/끝 위치로 계산 | 구간 개념을 더 명확하게 볼 수 있음 |
| 수학식으로 대각선 바로 찾기 | 더 간결할 수 있지만 구현 난도가 조금 올라감 |
