# 🧩 BOJ 2563 - 색종이

## 📌 문제 정보
- **난이도**: Silver 5
- **링크**: https://www.acmicpc.net/problem/2563

---

## 📝 문제 요약
- 색종이의 개수 `N`이 주어진다
- 각 색종이는 크기가 **10 × 10**이다
- 색종이를 도화지 위에 붙일 때, 서로 겹칠 수도 있다
- 색종이가 붙은 전체 넓이를 구한다
- 겹치는 부분은 여러 번 세지 않고 **한 번만 계산**해야 한다

---

## 💡 핵심 아이디어
- 100 × 100 크기의 도화지를 2차원 리스트로 만든다
- 색종이가 붙는 위치마다 **10 × 10 범위를 1로 표시**한다
- 이미 1인 곳을 다시 1로 표시해도 괜찮으므로, 겹치는 부분은 자동으로 한 번만 처리된다
- 마지막에 전체 배열에서 1의 개수를 세면 색종이가 덮은 넓이가 된다

---

## 🧠 접근 과정
- 처음 생각:
  - 색종이마다 넓이 100을 더하면 될 것 같지만, 겹치는 부분이 있어서 그렇게 하면 안 된다
- 핵심:
  - 도화지를 직접 2차원 배열로 표현하고, 색종이가 덮는 칸을 표시하면 된다
- 주의:
  - 겹치는 부분은 중복 계산하면 안 된다
- 해결:
  - 색종이가 덮는 칸을 `1`로 표시하고, 마지막에 `1`의 개수만 세면 된다

---

## 🛠 사용한 것
- 2차원 리스트
- 이중 `for`문
- `range()`
- `sum()`
- `set()`
- 좌표 순회

---

## ✅ 내가 배운 점
- 단순히 넓이를 더하는 것이 아니라, **겹치는 영역을 어떻게 처리할지** 생각해보는 계기가 되었다
- 2차원 배열을 사용하면 도화지를 직접 표현할 수 있어서 문제를 직관적으로 해결할 수 있다
- 어떤 좌표 범위를 직접 순회하면서 값을 표시하는 방식을 구현하는 법을 배웠다
- 이미 방문한 칸을 다시 방문해도 같은 값으로 처리하면, 겹치는 영역을 자연스럽게 해결할 수 있다
- 같은 문제도 2차원 배열 방식, `set`으로 좌표를 저장하는 방식처럼 여러 방법으로 풀 수 있다는 것을 배웠다
- 좌표를 `(x, y)`처럼 **tuple로 묶어서 관리**하면 한 개의 위치 정보처럼 다루기 편하다는 점도 익혔다
- `set`에 변경 가능한 자료형(`list`)은 넣을 수 없고, 변경 불가능한 자료형(`tuple`)만 넣을 수 있다는 점을 배웠다

---

## 🔄 여러 구현 방식 비교

| 방법 | 특징 |
|------|------|
| 2차원 배열에 표시 ✅ | 가장 기본적이고 직관적 |
| `set`으로 좌표 저장 | 겹치는 좌표를 자동으로 한 번만 처리 가능 |
| 2차원 배열 + 전체 합 계산 | 구현은 비슷하지만 결과 계산이 더 간단함 |
